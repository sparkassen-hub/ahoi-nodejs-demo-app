1. Neuen Benutzer in AHOI anlegen

1. Create a new user in AHOI

Warum eigentlich einen Benutzer anlegen, wenn man sich bereits bei AHOI registriert hat?

Question here is: why create a user if you have already registered with AHOI?

Man muss unterscheiden zwischen der Anwendung und den Benutzern der Anwendung.

It is necessary to distinguish between the application and the users of the application.

Die Anwendung kommuniziert mit der AHOI API, legt neue Benutzer bei AHOI an und fragt Bankdaten für die Benutzer ab. Damit eine Anwendung auf die AHOI API zugreifen kann, muss sie sich gegenüber AHOI mit clientSecret und clientId authentifizieren. Die Zugangsdaten dafür erhält man durch die initiale Registrierung eines (Demo-)Accounts bei AHOI.

The application communicates with the AHOI API, creates new users at AHOI and fetches bank data for users. In order for an application to be able to access the AHOI API, it must authenticate with clientSecret and clientId before. The access data is obtained from the initial registration of a (demo) account with AHOI.

Für jeden Benutzer der Anwendung, für den Bankdaten über die AHOI API abgerufen werden sollen, muss die Anwendung einen neuen Benutzer bei AHOI angelegen. Die AHOI API liefert daraufhin eine eindeutige Benutzerkennung (InstallationId genannt) zurück. Mit dieser werden von dem Benutzer freigegebene Bankzugänge verknüpft.

For each user of the application for whom bank data should be retrieved via the AHOI API the application must create a new user at AHOI. The AHOI API then returns a unique user ID (called InstallationId). The granted bank accesses of an user will be linked in later steps with his AHOI user ID (installationid).

Wichtig: AHOI bietet keine Benutzerverwaltung an. Mit der Benutzerkennung sind keine Benutzerdaten, wie z.B. eine E-Mail-Adresse, verbunden. Vielmehr sind nur die Bankzugänge und -Daten mit der Benutzerkennung verbunden.

Note: AHOI does not offer a user management tool. The user ID does not contain any user data, such as an email address. Actually only the granted bank accesses are linked with the user ID.

Die Verknüpfung zwischen einem Benutzer und einer Benutzerkennung (InstallationId), muss die Anwendung selber vornehmen.

The linkage between a user and a (AHOI) user ID must be made by the application.

Der übliche Weg dafür ist:

The common way for that is:

  • Ein Benutzer registriert sich bei der Anwendung mit Zugangsdaten wie Email und Passwort
  • A user registers at the application with access data such as Email and Password
  • Die Anwendung speichert die Benutzerdaten in der eigenen Benutzerverwaltung
  • The application stores the user data in its own user management tool
  • Die Anwendung registriert einen neuen Benutzer bei AHOI über die AHOI API
  • The application registers a new user at AHOI via the AHOI API
  • AHOI liefert eine eindeutige Benutzerkennung (InstallationId) zurück. Diese wird von der Anwendung mit dem Benutzer verknüpft und in der Benutzerverwaltung gespeichert
  • AHOI returns a unique user ID (InstallationId). This is linked to the user by the application and stored in the user management tool
  • Ruft ein Benutzer über die Anwendung dann z.B. Kontoumsätze ab, ermittelt die Anwendung zuerst die AHOI-Benutzerkennung (InstallationId) für den Benutzer in der Benutzerverwaltung und authentifiziert dann den Benutzer gegenüber AHOI mit der Benutzerkennung
  • On subsequent calls by the user, e.g. to get account transactions displayed, the application first queries the AHOI user ID (InstallationId) for the user in his user mangement tool and then authenticates the user to the AHOI API with the user ID

Diese Anwendung bietet eine sehr einfache Benutzerverwaltung, über die Benutzer authentifiziert und deren InstallationId gespeichert wird.

This application has a very simple user management tool that allows to authenticate users and store their AHOI installationid
